## 3. 创建和保护您的 API 密钥

在此处创建和管理您的 API 密钥：

👉 [https://kie.ai/api-key](https://kie.ai/api-key)

**重要的安全提示：**

* **切勿在前端代码中暴露您的 API 密钥**（浏览器、移动应用、公共仓库）。
* 将您的 API 密钥视为机密。

为了帮助保护您的使用，我们提供：

* **每个密钥的速率限制**（每小时、每日和总使用量上限）
* **IP 白名单**支持，仅允许已批准的服务器 IP 访问 API

这些功能有助于防止意外过度使用和未经授权的请求。

***

## 4. 必需的请求头

每个 API 请求**必须**包含正确的请求头：

```http  theme={null}
Authorization: Bearer <YOUR_API_KEY>
Content-Type: application/json
```

如果缺少或错误这些请求头，您可能会收到：

```json  theme={null}
{"code":401,"msg":"You do not have access permissions"}
```

在调试身份验证问题时，请务必仔细检查您的请求头。

## 6. 数据保留策略

请注意我们的保留规则：

* **生成的媒体文件**：存储 14 天，然后自动删除
* **日志记录**（文本/元数据）：存储 2 个月，然后自动删除

如果您需要长期访问，请确保及时下载并在您这边存储结果。

***

## 7. 异步任务模型

KIE 上的所有生成任务都是异步的。

成功的请求返回：

* **HTTP 200**
* 一个 `task_id`

<Warning>
  200 响应仅表示任务已创建，而非已完成。
</Warning>

要获取最终结果，您必须：

* 在请求中提供回调（webhook）URL，或
* 使用 `task_id` 主动轮询"查询记录信息" API

***

## 8. 速率限制和并发

默认情况下，我们应用以下限制：

* 每 10 秒最多 20 个新的生成请求
* 这通常允许 100+ 个并发运行的任务
* 限制按账户应用

如果您超过限制：

* 请求将被拒绝，返回 HTTP 429
* 被拒绝的请求不会进入队列

对于大多数用户来说，这已经足够了。
如果您持续遇到 429 错误，可以联系支持以请求更高的限制 — 审批会谨慎处理。
